# Safari 5 Extensions [HD] (@mkbhd, 2010-06-27)

## Summary
A direct follow-up to the 2010-06-08 same-day Safari 5 launch video: walks through
enabling Safari's new (then-hidden, developer-menu-gated) extension support and
installing an ad-blocking extension from a third-party directory, positioned against
Chrome and Firefox already having extensions/add-ons.

## Key claims
- [2010-06-27] **Frames Safari as catching up, not leading**: "Safari was the only
  main browser next to Internet Explorer... that's missing extensions" — an explicit
  competitive-gap framing rather than neutral feature coverage.
- [2010-06-27] **Restates the Chrome preference from the 2010-06-08 video, with a
  reason attached**: "I've been using Chrome for a while... I think that extensions
  will be the thing that finally hooks me [on Safari]" — ties platform switching
  explicitly to a single missing feature.
- [2010-06-27] **Step-by-step enablement documented**: Preferences → "Show Develop
  menu in menu bar" → Develop menu → "Enable Extensions" — the extension is then
  installed from a third-party (Tumblr) directory rather than an official Apple
  store, since Safari had no built-in extension gallery at this point.

## Notable verbatim quotes (Marques — voice data)
> "Safari was the only main browser next to Internet Explorer — which is obviously
> not something we want to look up to — that's missing extensions of some sort."

> "I've been using Chrome for a while... I kind of think that extensions will be the
> thing that finally hooks me."
