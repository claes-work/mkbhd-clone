# HD Tutorial: Boot Your Computer Faster (@mkbhd, 2009-02-04)

## Summary
Origin-era Vista boot-optimization screencast built around `msconfig` (System
Configuration). Marques covers the General (selective startup), Boot (lowering the boot
timeout; enabling No-GUI boot for a "blazingly fast" start), Services, and Startup tabs,
disabling components he doesn't need (e.g. Tablet PC input service on a non-tablet). Two
early-Marques tells stand out: his recurring **"low but not zero"** guidance (timeout
between 3 and 999, set to 3), and an aesthetics-over-raw-speed preference — he **keeps the
GUI boot even though No-GUI is faster because he likes the graphical loading screen**. He
points viewers to external websites for which services are safe to disable rather than
overclaiming his own authority.

## Key claims
- [2009-02-04] **How-to:** speed up Vista boot via `msconfig` — selective startup, lower
  boot timeout, and disabling unneeded Services/Startup items.
- [2009-02-04] **Boot timeout (dated):** many people have a 30-second timeout; it can be
  set between **3 and 999** seconds — he sets **3** ("don't make it zero").
- [2009-02-04] **Aesthetic preference (dated):** No-GUI boot loads "blazingly fast," but
  he **keeps the GUI boot because he likes the graphical loading screen** — an early
  sign he weighs experience/polish over raw numbers.
- [2009-02-04] **Epistemic humility:** defers to external sites on which services are
  safe to disable rather than asserting a definitive list.

## Notable verbatim quotes (Marques — voice data)
> "a no GUI boot will load absolutely blazingly fast but I actually like the GUI boot so I'm going to leave it"

> "there are tons of websites that will teach you which … services not to basically not to uncheck"
