# HD Tutorial: Make Firefox Faster (@mkbhd, 2009-02-02)

## Summary
Origin-era how-to for tuning **Firefox 3.0.5** via `about:config`: enable HTTP pipelining
(`network.http.pipelining` = true, `…max-requests` = 200, `…proxy.pipelining` = true) and
raise `network.http.max-persistent-connections-per-server` to 80. He notes the returns
diminish ("you can't set it to a trillion"). He also mentions the **FasterFox** add-on but
shows it is incompatible with 3.0.5. Biographically notable: he states he is **migrating
from Safari to Firefox**, having used Safari as his primary browser until "the past few
days" — a dated early-browser-preference data point.

## Key claims
- [2009-02-02] **How-to:** in `about:config`, set `network.http.pipelining` true,
  `max-requests` to 200, `proxy.pipelining` true, and `max-persistent-connections-per-
  server` to 80 to speed Firefox 3.0.5.
- [2009-02-02] **Judgment on tuning:** the connection values hit diminishing returns —
  going arbitrarily high doesn't help.
- [2009-02-02] **Browser migration (biographical):** he had been using **Safari primarily**
  and was **slowly moving to Firefox** "for the past few days."
- [2009-02-02] **Compatibility note:** the **FasterFox** add-on works up to Firefox 3.0.3
  but not 3.0.5; recommends upgrading rather than staying on an old version for it.
